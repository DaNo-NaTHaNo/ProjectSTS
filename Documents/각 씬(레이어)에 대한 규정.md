# **메타 레이어(로비) 관련 규정**

## 각 화면에 대한 설명

* 로비 \- 게임의 최상위 레이어로,파티 및 덱 구성과 탐험 개시 버튼, 캠페인 진척 상황 및 캐릭터 습득 상황에 따른 분위기를 조성하는 화면 연출이 구현되어 있음.   
  * 인벤토리 \- 탐험을 통해 습득한 카드와 아이템을 열람하기 위한 하위 레이어로, 목록 확인의 용이성을 위해 필터링 기능과 정렬 기능이 구현되어 있음  
    * 인벤토리 리스트 영역 \- 카드/아이템의 2개 탭으로 이루어진 그리드형 리스트로, 필터링 기능과 정렬 기능이 구현된 영역  
    * 아이템 정보 영역 \- 리스트 내 카드 혹은 아이템을 클릭했을 때, 해당 소지품의 이름과 효과 설명문, 아트 리소스를 확인할 수 있게 표시되는 영역  
  * 파티 편성 \- 탐험을 통해 습득한 유닛과 카드, 아이템을 다음 탐험에서 활용하기 위해 장비하고 구성하는 화면으로, 습득한 유닛의 프로필과 장비한 카드 덱, 아이템을 확인할 수 있음  
    * 파티 배치 영역 \- 유닛 리스트 영역에서 유닛을 드래그해 위치시킬 수 있는 3개의 빈 칸. 해당 칸에 위치시킨 유닛이 월드맵을 탐험하는 파티 구성원으로, 전투 포트레이트와 장비한 아이템이 표시됨.  
    * 유닛 리스트 영역 \- 습득한 유닛이 모두 표시되는 영역으로, 각 유닛의 전투 포트레이트가 나열된 가로 형식의 리스트. 각 유닛을 클릭하면 ‘ 유닛 편집 영역’이 출현,  
    * 유닛 편집 영역 \- 캐릭터에 장비할 아이템과 덱의 카드를 편집하는 영역으로, 인벤토리 리스트 영역과 함께 아이템 장비 슬롯 2개+카드 덱 구성 영역으로 이루어짐.  
  * 캠페인 진척 상황 \- 현재까지 진행을 완료했거나 진행중인 메인 및 서브 캠페인의 내용을 확인하는 화면으로, 캠페인 이름 및 하위목표 리스트와 선택한 캠페인에 대한 상세 설명 영역으로 이루어짐  
    * 캠페인 리스트 영역 \- 현재 진행 중인 캠페인과 완료한 캠페인을 표시하는 리스트로, 다음의 2개 조작을 행할 수 있음  
      * 리스트 각 항목의 제목을 클릭할 시 캠페인 설명 화면 영역에 해당 캠페인에 대한 상세 정보를 표시  
      *  리스트 각 항목의 추적 버튼을 클릭할 시 외부의 캠페인 내비게이터에 해당 캠페인의 제목 및 현 단계에서 수행해야 하는 하위 목표를 표시  
    * 캠페인 설명 화면 영역 \- 리스트에서 선택한 캠페인의 이름, 설명, 각 하위 목표의 이름과 설명을 표시하는 영역으로, 해당 캠페인에서 이미 완료한 하위 목표는 아래쪽에 표시하여 다음으로 수행할 목표가 잘 보일 수 있게 함

## 유닛 및 파티 구성 관련 규정

* 아군의 각 유닛은 0\~1개의 스킬, 0\~2개의 아이템, 4\~6장의 카드 덱을 장비함  
  * 카드 덱에는 기본적으로 6장의 카드를 장비하며, 파티에 배치된 인원 수에 따라 탐험 개시 시 5번째\~6번째 카드가 덱에서 제외됨  
    * 파티 인원 수가 1명일 경우 제외하지 않으며, 2명일 경우 6번째 카드 1장, 3명일 경우 5\~6번째 카드 총 2장을 덱에서 제외함  
*  아군 파티는 최소 1명 \~ 최대 3명의 유닛으로 이루어지며, 편성 화면에 배치된 유닛 위치 슬롯에 부여된 번호(1, 2, 3번)에 따라 전투 씬에 배치됨  
  * 시나리오 상의 특별한 경우를 제외하고, 주인공 유닛이 파티에 반드시 포함되어야 함.

## 인벤토리, 카드 및 아이템 관련 규정

* 카드, 스킬 및 아이템에 대한 정의  
  * 카드 \- 전투 시 직접적으로 사용하게 되는 자원이자 장비품으로, 각 캐릭터의 스킬 발동을 위한 트리거 역할  
  * 스킬 \- 특정 조건을 만족 시 자동적으로 발동하는 강력한 카드로, 실질적인 콤보 구축과 전투 경험의 중심축이 되는 역할  
  * 아이템 \- 유닛의 스펙이나 사용 카드 및 스킬의 수치를 강화하는 역할  
* 유닛 장비 관련  
  * 모든 유닛은 자신의 속성과 같은 속성의 카드 및 Wild 속성의 카드를 덱에 포함할 수 있음  
    * Wild 속성의 유닛은 모든 속성의 카드를 덱에 포함할 수 있음  
  * 모든 유닛은 자신의 속성과 같은 속성의 스킬을 장비할 수 있음  
  * 모든 유닛은 기본적으로 모든 아이템을 장비할 수 있음  
* 인벤토리 관련  
  * 인벤토리의 각 아이템은 아래와 같이 정렬 및 필터링이 가능  
    * 이름, 속성, 레어도, 코스트, 소지 갯수 순으로 정렬  
    * 카드 인벤토리의 경우, 속성, 코스트, 효과 포맷 정보로 필터링  
    * 아이템 인벤토리의 경우, 효과 포맷, 효과 적용 대상 스탯, 소모형 아이템 여부로 필터링

# **스테이지(월드맵) 관련 규정**

시작점을 중심으로 방사형으로 퍼지는 그물형 노드 맵이 구성되어 있고, 시작점으로부터의 방위 및 거리에 따라 나눠진 구역 별 이벤트 배치 테이블에서 정해진 종류와 수량 만큼 이벤트를 무작위 배치하는 방식. (유사 오픈 월드)

캐릭터에 따라 행동력이 부여되어 있으며, 파티를 구성한 캐릭터의 행동력 합계만큼 이동할 수 있음. 행동력이 고갈될 시 스테이지 종료.

### **플레이어의 이동 관련 규정**

* 행동 규칙  
  * 플레이어는 시작점에서 출발, 자신이 위치한 칸과 이어진 노드라면 어느 방향으로도 이동 가능  
  * 이동한 노드에 이벤트가 배치되어 있을 경우, 행동력을 1 소모해 이벤트를 실행  
  * 이벤트 종료 후 플레이어의 행동력이 0일 경우, 귀환으로 인한 스테이지 완료 처리를 실행  
* 시야 범위  
  * 특정 구역에 처음 입장 시, 해당 구역의 전체 이벤트 파이프라인을 표시  
  * 플레이어가 위치한 구역 내 이벤트와 직접 연결되어 있는 노드를 표시  
  * 캠페인 목표 이벤트는 노드 파이프라인 표시 여부에 상관 없이 맵 상에 표시

### **이벤트 노드 배치 관련 규정**

* 이벤트 배치 패턴  
  * 구역 내 레벨 당 이벤트 갯수 : (레벨 당 칸 갯수/2) \~ (레벨 당 칸 갯수-레벨 수치)  
  * 노드가 배치된 레벨 n 칸에 인접한 레벨 n+1 칸에 다음 이벤트 노드를 배치

### **스테이지 완료 규정**

* 귀환 조건 일람  
  * 스테이지 완료  
    * “보스전” 이벤트를 완료  
    * 캠페인의 주 목표를 달성  
    * 아군 파티의 남은 행동력이 0  
    * 특정 이벤트(탈출구 등)를 통한 중도 귀환 (생존 보상)  
  * 스테이지 실패  
    * 아군 파티 내 모든 유닛의 HP가 0  
    * 특정 이벤트를 실패  
* 스테이지 완료 보상 책정 규칙  
  * 고정 보상 (수량 \= 1\~5개)  
    * 해당 스테이지 중 새로 획득한 아이템을 “완료한 이벤트 갯수/3”(올림 처리)개만큼 선택해 획득  
      * 새로 획득한 것 중 선택받지 않은 아이템은 로비로 인계되지 않고 제거됨  
    * 스테이지 실패 시 레어도 제한 존재 (Epic 아이템 획득 불가)  
  * 완료 보상 (수량 \= 1\~3개)  
    * 해당 구역에서 드랍 테이블 내 아이템 중 무작위로 추가 획득  
    * 다음 트리거를 통해 스테이지 완료 처리 시, 완료 보상 수량 3개  
      * 레벨 15 ”보스전” 이벤트를 완료  
      * 캠페인의 주 목표를 달성  
    * 다음 트리거를 통해 스테이지 완료 처리 시, 완료 보상 수량 1개  
      * 귀환 이벤트 완료를 통한 중도 귀환  
      * 아군 파티 행동력 0으로 인한 귀환  
  * 추가 보상 (수량 제한 없음)  
    * 해당 스테이지 중 획득한 추가 보상 갯수만큼, 드랍 테이블 내 아이템 중 무작위로 획득  
      * “엘리트 전투” 이벤트 완료 시 추가 보상 갯수 \+1  
      * 특정 사건 이벤트 완료 시 추가 보상 갯수 \+1  
      * 캐릭터 관련 비주얼 노벨 이벤트 완료 시 추가 보상 갯수 \+1  
    * 스테이지 실패 시에도 획득한 추가 보상 갯수만큼 수령 가능

# **전투 관련 규정**

## 전투 경험에서 어필하고자 하는 것

1. 전장 상황 및 카드 사용에 따라 자동으로 발동하는 캐릭터 스킬을 통한 콤보 설계  
2. 제한된 카드 매수를 통해, 미리 설계한 콤보가 자주, 확실하게 발동하게 하는 방향성  
3. 어려운 난이도의 쫀쫀한 전투 설계보다는, 약점 공략과 기믹 파훼, 스킬 콤보로 인한 호쾌한 경험 위주

## 전투의 기본 메커니즘

1. 전투 시작  
   파티 구성원의 장비 덱을 모아 전투 덱을 생성  
   파티 구성원의 보유 에너지를 모아 전투 에너지 기본값을 생성  
   이벤트 정보에 입력된 적 유닛을 생성  
   1. 턴 시작  
      * 플레이어의 손패가 4장 미만이라면, 4장이 되도록 전투 덱에서 카드를 드로우  
      * 플레이어의 현재 전투 에너지가 기본값 미만일 경우, 기본값이 되도록 회복  
      * 적 유닛 AI에 따라 해당 턴의 행동을 결정 후 이를 UI 상에 표시  
      * 턴 시작 시 발동 처리하는 AI나 상태이상 효과 존재 시, 이를 발동 처리  
   2. 플레이어 행동 페이즈  
      * 주어진 손패와 전투 에너지를 소모해 카드를 소모  
      * 아래와 같은 상황일 때 플레이어 행동 페이즈 종료  
        * 플레이어가 전투 에너지를 모두 소모했을 때  
        * 플레이어의 손패가 0장일 때  
        * 플레이어가 UI 상의 행동 종료 버튼을 클릭했을 때  
   3. 적 행동 행동 페이즈  
      * 턴 시작 시 결정된 행동을 실행  
      * ‘행동 불능’ 속성의 상태 이상일 경우 ‘행동 실패’ 실행  
   4. 턴 종료  
      * 턴 종료 시 발동 처리하는 AI나 상태이상 효과 존재 시, 이를 발동 처리  
      * 상태이상, AI 패턴 등의 턴 카운트를 진행  
2. 전투 종료  
   1. 다음과 같은 상황일 때, 전투 종료 처리  
      * 적 유닛이 0체일 때 (승리)  
      * 아군 유닛이 모두 HP 0일 때 (패배)  
      * 플레이어가 전투 포기 버튼을 클릭했을 때 (패배)  
      * 이벤트 상의 전투 종료 트리거가 발동했을 때 (승리)  
   2. 전투 승리로 이벤트 종료 시 해당 이벤트 결과창 표시 (패배 시 스테이지 실패 결과창 표시)

## 캐릭터 스킬 처리 규정

* 각 유닛이 장비한 스킬의 발동 조건이 충족되었을 경우, 즉시 해당 스킬의 효과를 처리  
* 같은 타이밍에 여러 유닛의 스킬 발동 조건이 충족되었을 경우, 아래와 같은 우선도로 효과를 처리  
  1. 아군 유닛\>보스 유닛\>일반 적 유닛의 순서대로 발동 및 처리  
  2. 아군 유닛이 여럿일 경우, 파티 편성 위치 번호(1\~3) 순서대로 발동 및 처리  
  3. 일반 적 유닛이 여럿일 경우, 적 유닛이 배치된 순서대로 발동 및 처리

## 효과의 동시 발동 시 순서 처리 규정

아래의 규칙을 1번부터 우선적으로 적용

1. 예외처리로 최우선 발동을 지시한 효과  
2. 유닛이 장비한 스킬의 효과\>이미 적용된 상태이상의 효과  
3. 버프 효과\>디버프 효과  
4. 아군이 발동한 효과\>적이 발동한 효과  
5. 예외처리로 최후 순서 발동을 지시한 효과

## 시나리오 연출을 위한 예외 처리

특정 전투 이벤트에 한해, 기획자가 배치한 트리거에 따라 아래와 같은 시퀀스를 강제로 실행할 수 있음

1. 비주얼 노벨 재생  
2. 전투용 컷인 연출 재생  
3. 파티 멤버 및 적 유닛의 스탯을 변경  
4. 적 유닛의 패턴을 변경  
5. 플레이어의 덱 및 손패의 카드를 조작  
6. 아군 유닛이 장비한 스킬을 변경

# **시나리오 관련 규정**

## 캠페인 관련 규정

* 캠페인은 아래와 같은 2가지 분류로 나뉘어짐  
  * 메인 캠페인 \- 게임의 시작부터 끝까지를 그리는 메인 스토리 라인  
  * 서브 캠페인 \- 메인 캠페인의 진행에 영향을 주지 않는 단편 스토리 라인의 집합  
*  각 캠페인에 설정된 목표 그룹의 완료 트리거 목표를 완수한 상태에서 메타 레이어에 진입했을 경우, 해당 캠페인을 클리어 처리한 후 보상 지급  
  * 클리어 후 연출이 설정되어 있을 경우, 이를 재생한 다음 클리어 처리

## 비주얼 노벨 관련 규정

* 다음과 같은 경우, 각 데이터에 규정된 비주얼 노벨 JSON 파일을 VisualNovelPlayer 프리팹으로 재생  
  * 유닛이 ‘비주얼 노벨’ 타입의 이벤트가 배치된 칸에 위치했을 경우  
  * 캠페인 매니저에 의해 ‘비주얼 노벨 재생’이 호출되었을 경우  
  * 보스 및 이벤트 전투 시 특정 상황에 트리거되어 ‘비주얼 노벨 재생’이 호출되었을 경우  
* 비주얼 노벨 플레이어의 데이터 규격은 “비주얼 노벨 — JSON 데이터 처리 구조 & 전체 필드 정리.md” 문서에 맞춤

## 조우 이벤트 관련 규정

* 조우 이벤트의 기본적인 구조는 이하와 같음  
  * 설정된 전투 스테이지 진입 후 설정된 비주얼 노벨 데이터를 VisualNovelPlayer 재생  
  * 상기 비주얼 노벨 데이터를 재생 완료 후, 해당 데이터에 삽입된 선택지에 설정된 이벤트를 실행  
  * 조우 이벤트 진입 → 전투 스테이지 생성 → 초기 비주얼 노벨 재생 → 유저가 선택지를 선택 → 초기 비주얼 노벨 재생 완료 → 선택지에 따른 후속 이벤트 재생 → 후속 이벤트 재생 완료 후 이벤트 클리어 처리
