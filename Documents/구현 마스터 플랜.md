# ProjectStS — 게임 시스템 파이프라인 구현 마스터 플랜

## Plan: 전체 게임 시스템 파이프라인 단계별 구현

---

### 분석

#### 요청 요약
미구현 상태인 **Core 인프라 → 공용 데이터 모델 → 전투 시스템 → 스테이지(월드맵) → 메타 레이어(로비)** 전체 파이프라인의 단계별 구현 로드맵 수립.

#### 현재 상태
| 항목 | 상태 |
|------|------|
| 비주얼 노벨 시스템 | 완료 (`Assets/VisualNovel/`) |
| CsvUtility / CsvRowDefinitions | 존재하나 **데이터 구조 가이드와 불일치** |
| BaseRepository 패턴 | 존재 (`Assets/VisualNovel/Scripts/Repository/`) |
| `Assets/_Project/` 디렉터리 | 미생성 |
| Core 인프라 | 미구현 |
| 공용 Enum / 데이터 모델 / SO | 미구현 |
| 전투 시스템 | 미구현 |
| 스테이지(월드맵) | 미구현 |
| 메타 레이어(로비) | 미구현 |

#### 데이터 구조 가이드 vs 기존 코드 불일치 (Phase 1에서 해소)
1. **속성명**: 가이드=`Baton(불)`, CLAUDE.MD 용어사전=`Wand` → **가이드 기준 `Baton` 통일**
2. **속성명**: 카드 테이블=`Sol(빛)`, 나머지=`Sola(빛)` → **`Sola` 통일** (다수 기준)
3. **CardRow**: 가이드에 없는 `keywords`, `canUpgrade`, `upgradedCardId` 존재 / 가이드의 `cardEffectId`, `isDisposable` 누락
4. **StatusEffectRow**: `effectElement`, `isExpendable`, `expendCount`, `duration` 누락
5. **UnitRow**: `initialSkillId` 누락
6. **AIPatternRuleRow**: `speechLine`, `cutInEffect`, `zoomIn` 누락
7. **AIConditionRow**: `value` 타입 불일치 (가이드=string, 코드=float)

#### 참조 문서
- `CLAUDE.md`
- `Documents/기획 컨셉 및 게임플레이 개요.md`
- `Documents/각 씬(레이어)에 대한 규정.md`
- `Documents/데이터 구조 가이드/` 내 23개 CSV 전체
- 기존 코드: `CsvUtility.cs`, `CsvRowDefinitions.cs`, `BaseRepository.cs`, `VisualNovelPlayer.cs`

### 설계 — 아키텍처 총괄

```
[Unity Scenes]
BootScene → LobbyScene → StageScene → BattleScene

[Phase 0: Core]  부트스트래퍼, 서비스 로케이터, 씬 전환
       ↓
[Phase 1: Data]  공용 Enum, 데이터 모델 23종, SO, CSV 임포터
       ↓
[Phase 2: Battle]  턴 시스템, 카드, 스킬, 상태이상, AI, 전투 연출
       ↓
[Phase 3: Stage]  육각 그리드, 구역, 이벤트 배치, 탐험/행동력
       ↓
[Phase 4: Meta]  파티 편성, 인벤토리, 캠페인, 탐험 개시
       ↓
[Phase 5: Integration]  게임 루프 통합, 저장, VN 브릿지, 보상 정산
```

#### 네임스페이스 맵
```
ProjectStS.Core          — 부트스트래퍼, 서비스 로케이터, 씬 전환
ProjectStS.Data          — 공용 Enum, 데이터 모델, SO 정의
ProjectStS.Battle        — 전투 시스템 전체
ProjectStS.Battle.Card   — 카드/덱 서브시스템
ProjectStS.Battle.Skill  — 스킬 서브시스템
ProjectStS.Battle.StatusEffect — 상태이상 서브시스템
ProjectStS.Battle.AI     — 적 AI 서브시스템
ProjectStS.Stage         — 월드맵/스테이지 시스템
ProjectStS.Meta          — 로비(인벤토리, 파티 편성, 캠페인)
ProjectStS.UI            — 공용 UI 컴포넌트
ProjectStS.Utils         — 유틸리티, 확장 메서드
```

## Phase 2: Battle System

### Step 2-1: BattleManager (전투 총괄)
```
파일: Battle/BattleManager.cs
```
- 상태 머신: `Init → TurnStart → PlayerPhase → EnemyPhase → TurnEnd → BattleEnd`
- 이벤트: `OnBattleStart`, `OnTurnStart`, `OnTurnEnd`, `OnBattleEnd`
- 종료 조건: 적 전멸(승리), 아군 전멸(패배), 포기(패배), 이벤트 트리거(승리)

### Step 2-2: TurnManager (턴 관리)
```
파일: Battle/TurnSystem/TurnManager.cs
```
턴 흐름 (기획서 기준):
1. **턴 시작**: 손패 리필(→4장), 에너지 회복(→기본값), 적 AI 행동 결정 + UI 표시, 턴 시작 효과 발동
2. **플레이어 행동**: 카드 사용 / 종료 조건(에너지 0, 손패 0, 행동 종료 버튼)
3. **적 행동**: AI 결정 행동 실행, 행동 불능 시 실패
4. **턴 종료**: 턴 종료 효과 발동, 상태이상/AI 턴 카운트 진행

### Step 2-3: BattleUnit (런타임 유닛)
```
파일: Battle/BattleUnit.cs
```
- `UnitData` 참조(마스터), 런타임 상태: CurrentHP, MaxHP, CurrentEnergy, ElementType, PartyPosition
- 활성 상태이상 목록, 장비 스킬 참조
- 이벤트: `OnDamaged`, `OnHealed`, `OnDeath`, `OnStatusApplied`

### Step 2-4: DeckSystem (덱/손패/묘지)
```
파일: Battle/Card/DeckSystem.cs
```
- 전투 덱 생성: 파티원 장비 덱 합산(최대 18장), 인원수별 카드 제외
  - 1명: 제외 없음 / 2명: 6번째 제외 / 3명: 5~6번째 제외
- 드로우: 턴 시작 시 손패 4장까지 리필 (Retain — 미사용 카드 잔류)
- 카드 사용: 에너지 소모 → 효과 실행 → 묘지(일회용이면 제거)
- 자료구조: `Queue<CardData>`(덱) — 성능 가이드라인 준수

### Step 2-5: CardEffectProcessor (카드 효과)

```
파일: Battle/Card/CardEffectProcessor.cs
```

```
파일: Battle/DamageCalculator.cs
```

### Step 2-7: SkillSystem (스킬 발동)
```
파일: Battle/Skill/SkillSystem.cs
```
- 장비 스킬의 `triggerTarget` + `triggerStatus` 모니터링
- 조건 충족 시 즉시 효과 실행
- 동시 발동 우선순위 (기획서):
  1. 예외처리 최우선 2. 스킬>상태이상 3. 버프>디버프 4. 아군>적 5. 예외처리 최후
- 복수 유닛: 아군>보스>일반적, 아군끼리=파티 위치 순, 적끼리=배치 순
- 사용 제약: CoolDown, PerTurn, PerBattle

### Step 2-8: StatusEffectSystem (상태이상)
```
파일: Battle/StatusEffect/StatusEffectSystem.cs
```
- `ActiveStatusEffect`: StatusEffectData + 현재 스택 + 남은 턴
- 스택: isStackable=true → 누적(최대 maxStacks), false → duration 가산
- 소비형: isExpendable=true → 효과 적용 시 expendCount만큼 스택 소모
- `TriggerTiming`별 발동: TurnStart, TurnEnd, OnAttack, OnDamage
- 턴 종료 시 duration -1, 0이면 제거

### Step 2-9: EnemyAISystem (적 AI)
```
파일: Battle/AI/EnemyAISystem.cs
```
- AIPattern → AIPatternRule → AICondition 3단 구조
- 턴 시작 시 행동 결정:
  1. 패턴의 모든 룰 조회
  2. 각 룰의 컨디션 평가 (TurnCount, TurnMod, HpPercent 등)
  3. 만족하는 룰 중 priority 최고값 선택
  4. 해당 룰 없으면 defaultActionType 실행
- 행동 불능(Stun) 시 행동 실패

### Step 2-10: EnemyWaveSystem (웨이브)
```
파일: Battle/EnemyWaveSystem.cs
```

### Step 2-11: BattleEventTimeline (전투 중 시나리오 연출)
```
파일: Battle/BattleEventTimeline.cs
```
- `BattleTimelineData` 트리거 조건 평가

## 확인 사항

1. **Addressable 시스템**: 아트 리소스 경로가 "어드레서블 ID"로 기술되어 있으나, 현재 프로젝트에 Addressable 패키지 미포함. Phase 0에서 도입 여부 결정 필요 (Resources 사용 금지 규칙 준수를 위해 도입 권장).
2. **UI 프레임워크**: 본 계획은 로직 중심. UI 레이아웃/프리팹 설계는 별도 계획이 필요할 수 있음.
3. **오브젝트 풀링**: 성능 가이드라인에 따라 카드 UI, 전투 이펙트, 월드맵 노드에 풀링 적용 필요. Phase 0에서 공용 ObjectPool 유틸 준비 권장.
4. **CLAUDE.MD 용어사전 갱신**: Wand→Baton 변경사항 반영 필요.
