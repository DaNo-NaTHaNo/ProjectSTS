## 1. 개요

Unity 에디터에서 발생한 컴파일 에러를 2차에 걸쳐 수정하였다.

**1차 (커밋 1) — 런타임 에러 2건:**
1. **CS0104**: `StageState.cs`에서 `EventType`이 `ProjectStS.Data.EventType`과 `UnityEngine.EventType` 사이에서 모호
2. **Burst 어셈블리 resolve 실패**: `ProjectStS.Editor` 어셈블리를 런타임 빌드에서 찾을 수 없음

**2차 (커밋 2) — 에디터 에러 2건:**
3. **CsvUtility 미참조**: `CsvToSOImporter.cs`에서 기본 어셈블리의 `CsvUtility` 클래스에 접근 불가
4. **CS0104**: `CsvToSOImporter.cs`에서 동일한 `EventType` 모호성 에러

**3차 (커밋 3) — 어셈블리 간 타입 참조 에러 2건:**
5. **VN 타입 미참조**: `CsvUtility.cs`(ProjectStS.Utils)에서 기본 어셈블리의 `DialogueLine`, `EpisodePortrait`, `LocationPreset` 접근 불가
6. **Burst 어셈블리 resolve 실패 (2차 에러)**: 5번 컴파일 실패로 인한 연쇄 에러

## 2. 에러 분석 및 수정

### 2.1 EventType 네임스페이스 모호성 (CS0104)

**에러 메시지**:
```
Assets\_Project\Scripts\Runtime\Stage\StageState.cs(124,43): error CS0104:
'EventType' is an ambiguous reference between 'ProjectStS.Data.EventType' and 'UnityEngine.EventType'
```

**원인**:
- `StageState.cs`에서 `using UnityEngine;`과 `using ProjectStS.Data;`를 동시에 선언
- `EventType`이라는 이름이 양쪽 네임스페이스에 모두 존재
  - `ProjectStS.Data.EventType` — 프로젝트 커스텀 enum (VisualNovel, BattleNormal 등)
  - `UnityEngine.EventType` — Unity 내장 IMGUI 이벤트 타입 enum
- 메서드 시그니처 `public void RecordEventCompletion(EventType eventType)`에서 타입 이름을 단독 사용하여 컴파일러가 구별 불가

**수정 내용**:
- `StageState.cs`, `StageManager.cs`, `GameFlowController.cs` 3개 파일에 using alias 추가:
  ```csharp
  using EventType = ProjectStS.Data.EventType;
  ```
- `StageManager.cs`와 `GameFlowController.cs`는 현재 에러 미발생이나, 동일한 using 조합을 사용하므로 향후 동일 에러 예방을 위해 함께 적용

### 2.2 ProjectStS.Editor 어셈블리 resolve 실패

**에러 메시지**:
```
Failed to resolve assembly: 'ProjectStS.Editor, Version=0.0.0.0, Culture=neutral, PublicKeyToken=null'
```

**원인**:
- `ProjectStS.Editor.asmdef`에서 `autoReferenced: true`로 설정되어 있음
- Burst 컴파일러가 런타임 코드 빌드 시 자동으로 Editor 어셈블리를 참조하려 시도
- 그러나 `includePlatforms: ["Editor"]`로 에디터 전용이므로 런타임 빌드 대상에서는 해당 어셈블리를 찾을 수 없어 에러 발생

**수정 내용**:
- `ProjectStS.Editor.asmdef`의 `autoReferenced`를 `true` → `false`로 변경
- Editor 어셈블리는 에디터 전용이므로 자동 참조가 불필요하며, `false`로 변경해도 에디터 코드에 영향 없음

### 2.3 CsvUtility 어셈블리 참조 불가

**에러 메시지**:
```
CsvUtility — 컨텍스트 상 존재하지 않음 (CsvToSOImporter.cs)
```

**원인**:
- `CsvUtility.cs`와 `CsvRowDefinitions.cs`가 `Assets/Scripts/Utilities/` (기본 Assembly-CSharp)에 위치
- `CsvToSOImporter.cs`는 `ProjectStS.Editor` asmdef 어셈블리에 소속
- **asmdef 어셈블리는 기본 어셈블리(Assembly-CSharp)를 참조할 수 없음** — Unity의 구조적 제약

**수정 내용**:
- `CsvUtility.cs`와 `CsvRowDefinitions.cs`를 `Assets/_Project/Scripts/Runtime/Utils/`로 이동
- `.meta` 파일도 함께 이동하여 Unity GUID 보존
- `ProjectStS.Utils`는 `autoReferenced: true`이므로 기존 사용처(VisualNovel 에디터 3곳)에서도 정상 접근 가능
- `ProjectStS.Editor`는 이미 `ProjectStS.Utils`를 참조하므로 CsvToSOImporter에서 즉시 접근 가능
- 네임스페이스 변경 없음 (글로벌 네임스페이스 유지)

### 2.4 CsvToSOImporter.cs의 EventType 모호성 (CS0104)

**에러 메시지**:
```
CsvToSOImporter.cs: error CS0104: 'EventType' is an ambiguous reference between
'ProjectStS.Data.EventType' and 'UnityEngine.EventType'
```

**원인**:
- 2.1과 동일한 원인. `using UnityEngine;` + `using ProjectStS.Data;` 조합에서 `EventType` 충돌
- `ParseEnum<EventType>(row.eventType)` 호출 시 발생

**수정 내용**:
- `CsvToSOImporter.cs`에 using alias 추가:
  ```csharp
  using EventType = ProjectStS.Data.EventType;
  ```

### 2.5 CsvUtility의 VN 타입 참조 불가

**에러 메시지**:
```
CsvUtility.cs: DialogueLine, EpisodePortrait, LocationPreset — 컨텍스트 상 존재하지 않음
```

**원인**:
- 2차 수정에서 `CsvUtility.cs`를 기본 어셈블리 → `ProjectStS.Utils` 어셈블리로 이동
- CsvUtility 내부에 VN 타입(`DialogueLine`, `EpisodePortrait`, `LocationPreset`)을 참조하는 메서드 6개 존재
- 이 VN 타입들은 `Assets/VisualNovel/Scripts/Runtime/`에 위치하며 기본 어셈블리에 소속
- **asmdef 어셈블리(ProjectStS.Utils)는 기본 어셈블리의 타입을 참조할 수 없음**

**수정 내용**:
- CsvUtility를 게임 데이터 전용 / VN 전용으로 **분리**
- `CsvUtility.cs`(ProjectStS.Utils): VN 메서드 6개 제거, `ParseCsvLine`·`Escape`를 `public`으로 변경
- `CsvUtilityVN.cs`(기본 어셈블리) **신규 생성**: VN 전용 6개 메서드 이동, `CsvUtility.ParseCsvLine()` 등 호출로 헬퍼 재사용
- VN 에디터 3개 파일의 호출부를 `CsvUtility.` → `CsvUtilityVN.`으로 변경

### 2.6 Burst 어셈블리 resolve 실패 (2차 에러)

**원인**: 2.5의 컴파일 실패로 `ProjectStS.Utils` 어셈블리가 빌드되지 않아 Burst 컴파일러가 연쇄적으로 실패. 2.5 해결 시 자연 해소 예상.
